多表查询
多表查询分连接查询和子查询。
连接查询是将两个表连接起来形成一张表再进行查询。连接查询的实质是从 A 表中依次取一条记录与 B 表中的每一条记录进行连接，生成一张新表，再在新表中进行查询。连接而成的新表中的每一条记录都包含两张表中的所有字段。不设置连接条件的连接查询生成的新表中的记录数是两张表记录数的乘积。
连接查询分内连接与外连接查询。外连接又可分为左外连接和右外连接和全外连接三种。
一、表之间的连接
1、内连接查询语法
内连接用于把两个表连接成第三个表，在第三个表中仅包含那些满足连接条件的记录行。
语法格式：
Select 表名。列名，表名。列名，……
From

Where <查询条件>
也可以写成简便形式：
Select 表 1. 列 1，表 2. 列 2 ……
From 表 1，表 2
Where 表 1. 列 = 表 2. 列
例 1、将 xs,cj 表进行内连接查询输出所有记录
Select * from xs inner join cj
连接查询默认是内部连接，所以内连接中的 inner join 可省略
简写成 select * from xs,cj
例 2、将 xs,cj 表按学号相等进行内连接，查询输出所有记录
Select * from xs inner join cj on xs.xs_id=cj.xs_id;
也可以写成
Select from xs,cj where xs.xs_id=cj.xs_id;
例 3、将 xs,cj 表按学号相等进行内连接，查询输出学生表中的学号、姓名，成绩表中的学号和成绩等字段
Select xs.xs_id,xs_name,xs.xs_id,cj from xs,cj where xs.xs_id=cj.xs_id;
注意：当某个字段在两个表中的名字相同时，要指定表名。
如：xs.xs_id 和 cj.xs_id
例 4、在 xsgl 数据库中查询出每个同学的每门课程的成绩；
Select kc_name,cj. from kc,cj where kc.kc_id=cj.kc_id; // 使用内连接查询
注意：cj.* 表示输出 cj 表中的所有字段；
例 1：
在学生管理数据库中查询出学生的姓名、课程名和成绩；
Select xs. 姓名，kc. 课程名，cj. 成绩 from xs,kc,cj where xs. 学号 = cj. 学号 and kc. 课程号 = cj. 课程号
【注意】
当要输出的字段只有一个表中有时，字段前的表名可省略。
以上查询也可以写成:
Select 姓名，课程名，成绩 from xs , kc , cj where xs. 学号 = cj. 学号 and kc. 课程号 = cj. 课程号
2、外连接
外连接查询只限制其中一个表的行，而不限制另外一个表中的行。外连接分为左外连接、右外连接和全外连接三种，外连接只限于两个表中。
Left outer join ：包括了左表中的全部行和右边表中满足连接条件的行；
Select xs.* , cj.* from xs left outer join cj on xs. 学号 = cj. 学号
也可写成：
Select * from xs left outer join on xs. 学号 = cj. 学号
左连接有 31 条记录
Right outer join ：包括了右表中的全部行，和左边表中满足连接条件的行；
Select xs.* , cj.* from xs right outer join cj on xs. 学号 = cj. 学号

Full outer join ：包括了左表和右表中的所有行；

全连接，与内连接一样的语法，只不过不需要设置查询条件。如：
Select * from xs , cj
关于外连接的几点说明：
A、左外连接合并两个表中满足连接条件的行加上在 jion 子句中指定的左表中不满足条件的行。
B、不满足连接条件的行在结果中显示为 Null；
C、全外连接中，参加连接的两个表中的每一条记录都与另一个表中的每一条记录连接，其结果集的实记录数是两张表记录数的积。
D、左右表是相对的，在语句中先出现的表称为左表，后出现的表称为右表。
3、多表连接查询的实质
多表连接查询实际上是根据查询要求先将两个表连接起业，形成一张新表，再在新表中查询出满足条件的记录。
练习：在 xsgl 数据库中查询出无人参加考试的这些课程的课程号、课程名和学分；
Select kc.kc_id,kc_name,kc_xf from kc left outer join cj on kc.kc_id=cj.kc_id where cj.kc_id is null;
Select kc.kc_id,kc_name,kc_xf from kc where kc_id not in(select kc_id from cj);
二、子查询
在查询中，当一个查询的结果可以作为另一个查询的条件时，这个查询称作它的子查询（里层查询）。SQL 允许 Select 多层嵌套使用，用来表示复杂的查询。
子查询通常与 IN 、Exist 及比较运算符给合起来用。
例 2：
在 xsgl 数据库中查找选修了课程号为 206 的课程的学生的姓名、学号。
Select xs_id,xs_name from xs where xs_id in(select xs_id from cj where kc_id= "206 ");
【说明】
在执行包含子查询的 select 语句时，系统先执行子查询，产生一个结果表，再执行查询。

练习：
在学生管理数据库查询出参加了离散数学考试的学生的姓名。

扩展练习：
在学生管理数据库查询出末参加了离散数学考试的学生的姓名。

例 3：
在学生表中查找出比所有计算机专业的学生都要大的学生学号、姓名、专业名和出生日期。
